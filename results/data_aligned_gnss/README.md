# data aligned GNSS bag

This directory contains derived artifacts for `/home/zcl/data.bag`.

## Generated topic

`data_with_aligned_gnss.bag` keeps all original messages and adds:

- `/ekf/aligned_gnss/fix`: `sensor_msgs/NavSatFix`, 1442 messages

The topic is generated by locally aligning raw GNSS samples to
`/mavros/odometry/out` with a sliding 2D rigid fit, then applying an odom-guided
continuity constraint so local window changes do not introduce visible GNSS path
steps.

Generation command:

```bash
python3 dataset_tools/align_gnss_to_odom_bag.py \
  --input-bag /home/zcl/data.bag \
  --output-bag results/data_aligned_gnss/data_with_aligned_gnss.bag \
  --gnss-topic /mavros/global_position/raw/fix \
  --odom-topic /mavros/odometry/out \
  --output-topic /ekf/aligned_gnss/fix \
  --window-s 30 \
  --min-pairs 20 \
  --max-step-correction 0.10 \
  --max-z-step-correction 0.08 \
  --covariance-xy 4.0 \
  --covariance-z 9.0
```

Observed generation residual:

- local fit mean: 0.054 m
- local fit P95: 0.106 m
- aligned GNSS vs odom: mean 0.070 m, P95 0.177 m, max 0.374 m
- aligned GNSS step-vs-odom step error: P95 0.100 m, max 0.100 m
- aligned GNSS z step: P95 0.107 m, max 0.180 m
- aligned GNSS z-vs-odom z-step error: P95 0.080 m, max 0.080 m

The latest machine-readable result is
`results/data_aligned_gnss/data_smooth_benchmark.json`.

## Important limitation

`/ekf/aligned_gnss/fix` is a derived debug/fusion input built using odom. It is
useful for inspecting GNSS participation and preventing frame-mismatch artifacts
in this recording, but it is not an independent GNSS truth source for accuracy
evaluation.
